Work-in-progress (WIP) Template & Supporting Files

Summary: These files track the lifecycle of design development programs, and their corresponding styles. Through status codes noted by the user at each stage, the data tabs pull useful statistics that show a summary of the category, and of the department as a whole. Statistics gathered show if there are trends that need to be examined, and more generally if the season is on track to reach adoption rate goals set by previous seasons. The supporting documents are used to summarize data at significant points along the development timeline, and eventually show the overall outcome and success of the season.

  • WIP Tab

    The WIP Tab is where all data is entered by the user. White lines are the header lines for each program, and each colored line below it represents an actual, physical sample within that program.

  • Label & Agenda Maker Tab

    The Label & Agenda Maker Tab is a tool to format labels by concatenating cells for use with photos, printed labels, and agenda lists.

  • List Maker Tab

    The List Maker Tab is a tool that works to reorder information from the WIP, to be input into the supporting documents.

  • Data Tabs

    The Data Tabs show thousands of program and style data points, broken down by various category perspectives. Tabs are by Class, by Vendor, and by Designer. There is also a Stage and Time data tab which breaks down adoption, dropped, and amended statistics by timeframe in development. The Stage and Time tab also shows average sampling time for each category, at each development timeframe, and the overall average for both class and vendor.

  • Code Key Tabs

    The Code Key Tab is a referene point for drop-down lists used on the WIP tab, as well as a key for explaining the use of each program or style code.

  • Macros

    Macros are used to automate the transfer of data from the WIP Tab, to the List making tab to be sorted, and from there input into the one of the supporting documents to show sample status, or results. Macros are also used to quickly make labels for any styles that have arrived that day, or to arrange a full agenda for an upcoming meeting.

Supporting Files

Working Schedule & Estimates File

  • Category Tabs

    The Category Tabs are used as the space to place sorted program data from the WIP to populate the summary charts.

  • Totals Tabs

    The Totals Tab summarizes all category tabs to give an overall status update for what will be present to review at meetings.

  • Macros

    Macros in the Working Shcedule & Estimates file are used to maintain the conditional formatting for all tabs.

Results File

  • Category Tabs

    The Category Tabs are used as the space to place sorted program data from the WIP to populate the results summary charts.

  • Totals Tabs

    The Totals Tab summarizes all category tabs to show the final outcome of the development season.

  • Macros

    Results Macros are used to maintain the conditional formatting, which shows if a target adoption rate was reached or missed.
